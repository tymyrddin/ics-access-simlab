# Shared zone networks (legacy directory)

This directory used to hold a generated `docker-compose.yml` that created the
six lab zone networks as docker user-defined bridges. The clab rewrite
replaced that with real host-side Linux bridges declared as `kind: bridge`
nodes inside each per-zone topology under [`clab/`](../../clab/).

`infrastructure/clab-up.sh` (also generated) creates the bridges with
`sudo ip link add ... type bridge` before `containerlab deploy` runs.
`clab-down.sh` removes them. There is no docker-compose stack here any
more; `generate.py` actively cleans up a stale file if a previous run
left one behind.

## Startup order

The orchestration is in `infrastructure/clab-up.sh` (generated by
`orchestrator/generate.py`). Roughly:

1. Pre-create the six host bridges via `sudo ip link add`.
2. Build `clab-router` (FRR + iptables + sshd image) and `lab-mysql8`.
3. `containerlab deploy` each per-zone topology.

`clab-down.sh` reverses this. See [`clab/README.md`](../../clab/README.md)
for the full clab story.
